## `$ uptime` — Engineering Principles

```
[✓] Design for failure        — assume everything will break
[✓] Isolate everything        — one user's data never touches another's
[✓] Kill-switch first         — build the off-switch before you flip the on-switch
[✓] Optimize later            — make it work, then make it fast
[✓] Observability first       — logs, metrics, traces from day one
[✓] Security by default       — never bolt it on later
[✓] Keep it simple            — complexity is the enemy of reliability
```
